2) Upgrade from unweighted median to freshness/source-weighted pricing
Your contract already computes freshness weights but says they’re advisory; the core median is still unweighted.

A 2-day-old PVRPV signal and a 45-day-old static fallback should not influence the result equally.

3) Replace hand-tuned premiums with calibration tables per segment

Issues I’d fix next (highest priority first)
React side effect during render in ForwardDemandPanel

Calibration timestamp robustness
new Date(snap.metadata.run_at) is assumed valid; if malformed, daysBetween can produce invalid math. Add a validity guard and return suppression reason like snapshot_invalid_timestamp. 
track-applied idempotency/semantics

Lifecycle reset for tracking cache
