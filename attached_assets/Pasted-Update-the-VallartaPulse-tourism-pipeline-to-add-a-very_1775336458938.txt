Update the VallartaPulse tourism pipeline to add a very lightweight “current month passenger estimate” layer on top of the existing GAP monthly scraper. Do not over-engineer this. This is not a mission-critical feature and does not need airline-grade precision. The goal is simply to show a reasonable current-month estimate before the official GAP monthly total is published.

What to build:

1. Keep the existing GAP scraper exactly as the source of truth for finalized historical monthly PVR passenger totals.

2. Add a simple estimator that calculates:
   - current_month_estimated_passengers
   - current_month_estimated_daily_average
   - estimated_vs_same_month_last_year_pct
   - estimate_status = "estimated" for the open month and "official" for closed months once GAP data is available

3. Estimation logic should be intentionally simple:
   - Use the most recent official GAP monthly PVR total from the prior month
   - Use the same month last year from GAP
   - Use day-of-month pacing to estimate the current month
   - Blend two simple methods:
     A. Prior month daily run-rate method
        prior_month_daily_avg = prior_month_total / days_in_prior_month
        run_rate_estimate = prior_month_daily_avg * days_elapsed_in_current_month
     B. Same month last year pacing method
        same_month_last_year_daily_avg = same_month_last_year_total / days_in_same_month_last_year
        yoy_paced_estimate = same_month_last_year_daily_avg * days_elapsed_in_current_month
   - Final elapsed-month estimate:
        elapsed_estimate = 0.5 * run_rate_estimate + 0.5 * yoy_paced_estimate
   - Final full-month projected estimate:
        projected_full_month = elapsed_estimate / (days_elapsed_in_current_month / days_in_current_month)

4. Add a seasonality adjustment table in code, but keep it modest and editable. Use a plain object keyed by month number:
   {
     1: 1.08,
     2: 1.05,
     3: 1.10,
     4: 1.02,
     5: 0.96,
     6: 0.90,
     7: 0.88,
     8: 0.89,
     9: 0.87,
     10: 0.95,
     11: 1.00,
     12: 1.15
   }
   Apply this only as a light multiplier to the projection relative to neutral month 1.00. Cap the effect so it cannot swing the estimate by more than plus or minus 15%.

5. Add a confidence label:
   - days elapsed 1-7 = "low"
   - days elapsed 8-20 = "medium"
   - days elapsed 21+ = "high"
